User tokens authenticate as your user account and inherit your current Ghostwriter permissions. Use them for personal access. Service tokens authenticate as non-human service principals and use only the permissions assigned to the token. Use these for automation and integrations.

Tokens are shown once when created or regenerated. Expiration dates cannot be further out than {{ general_config.token_max_lifetime_days }} days. Shortening expiry keeps the same token. Extending expiry {% if general_config.token_extend_requires_rotation %}rotates the token and immediately invalidates the old value{% else %} keeps the same token unless an admin enables forced rotation{% endif %}. Expired tokens must have a new expiration date set before regeneration. {% if not general_config.token_extend_requires_rotation %} Regenerate after extending expiry to rotate the credential.{% endif %}
